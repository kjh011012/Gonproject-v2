너는 “Figma Make 프롬프트 작성 전문가(프로덕트 디자이너 + UX 아키텍트)”다.
아래 요구사항을 기반으로, 내가 Figma Make에 그대로 붙여넣을 수 있는 “최종 Figma Make 프롬프트”를 작성해줘.
반드시 한국어로만 작성하고, 결과는 복사/붙여넣기 바로 가능한 형태로 구조화해줘.

========================
0) 프로젝트 개요 / 목표
========================
프로젝트: “횡성의료사회적협동조합” 웹사이트 + 관리자 운영 OS(Admin) 통합 설계

목표:
- 사용자(비회원/조합원)가 “조합 소개 → 조합원 가입 유도 → 서비스 신청/접수 → 커뮤니티/공지/문의”까지 한 흐름으로 편리하게 이용하도록 만든다.
- 관리자는 운영 OS에서 “서비스 신청/접수 상태”를 캘린더 + 리스트로 자동화 관리하고,
  신청 건마다 진행 단계(진행 척도/타임라인)를 매우 상세히 추적/확인/업데이트할 수 있어야 한다.
- 공지/문의/커뮤니티 운영 기능(등록·수정·삭제·답글/처리상태/관리자 권한)을 완비한다.
- 전체 UI 톤은 “따뜻한 돌봄/치유/신뢰(의료·복지·지역 공동체)”이며, 과도한 테크/사이버 감성은 금지한다.
- 산출물: Figma Make에서 생성 가능한 수준의 화면/컴포넌트/상태/권한/데이터 구조를 포함한 프롬프트

================================
1) 사용자 역할(권한) 정의
================================
A. 비회원(방문자)
- 홈페이지 열람(소개/서비스/조합원 혜택/후기/공지 일부/문의 작성 가능)
- 조합원 가입 CTA로 유도
- 서비스 신청은 기본적으로 “조합원 가입 후” 가능 (단, 서비스 정책에 따라 비회원 신청 옵션도 확장 가능)

B. 조합원(로그인 사용자)
- 내 프로필/조합원 상태/가입 정보 확인
- 서비스 신청 및 내 신청/접수 내역 조회
- 신청 건 진행 단계(접수/상담/배정/일정확정/진행/완료) 타임라인 확인
- 커뮤니티 글/댓글 작성 및 자기 글 수정/삭제
- 문의 작성 및 답변 확인
- 공지사항 열람(전체/조합원 전용 구분)

C. 관리자(운영자)
- 서비스 카탈로그(재택의료/마을 돌봄/지역 건강공동체) 생성/수정/비활성화
- 신청/접수 전건 관리:
  - 캘린더 뷰(방문 일정/배정 인력/지역/상태)
  - 리스트 뷰(필터/정렬/검색/일괄처리)
- 신청 건 진행 단계(진행 척도) 템플릿 정의/편집/상태 변경/담당자 배정
- 공지사항 등록/수정/삭제 (유형: 전체/조합원/긴급)
- 문의사항: 답글/상태(대기/처리중/완료)/템플릿 답변/담당자 배정
- 커뮤니티 운영: 게시글/댓글 관리(숨김/삭제/신고 처리)
- 통계/대시보드: 신청량, 완료율, 평균 처리시간, 서비스별 수요, 지역별 분포
- 권한 관리(역할 세분화: 슈퍼관리자/상담담당/방문일정담당 등)

========================================
2) 서비스 도메인(핵심 3종) 및 신청 흐름
========================================
서비스 카테고리:
1) 재택의료: 방문진료, 만성질환 관리, 재활/간호 연계 등(확장 가능)
2) 마을 돌봄서비스: 돌봄 방문, 생활지원, 정서지원, 건강 체크 등
3) 지역 건강공동체 프로그램: 건강교육/운동/식생활/모임/예방 중심

공통 신청/접수 흐름:
서비스 선택 → 신청서 작성 → 제출 → 접수 완료(알림)
→ 상담/확인 → 배정(담당자/팀) → 일정 확정(캘린더 이벤트 생성)
→ 방문/프로그램 진행 → 완료 → 만족도/후기(옵션)

신청서 기본 필드:
- 신청자(조합원) 정보 자동채움: 이름/연락처/주소
- 서비스 유형(재택의료/돌봄/프로그램)
- 희망 일정(복수 후보) + 시간대
- 대상자 정보(본인/가족/기타) + 생년월일(선택) + 특이사항
- 증상/요청 내용(텍스트)
- 개인정보 동의(필수)

관리자가 관리하는 핵심 필드:
- 상태(status): 신청됨 → 접수확인 → 상담중 → 배정완료 → 일정확정 → 진행중 → 완료 → 보류/취소
- 담당자/팀 배정
- 일정(캘린더 이벤트)
- 진행 척도(체크리스트/타임라인):
  예) “전화상담 완료 / 서류확인 / 방문인력 배정 / 방문완료 / 기록작성 / 후속안내”

3) 사이트 전체 IA(페이지 구조)
==============================
3-1) 공개 홈페이지(비회원/조합원 공통)
- 메인(강력 소개 랜딩)
  - 메인 히어로: 아래 “WebGL Hero” 분위기(움직이는 추상 배경)로 표현
  - 우리의 돌봄 가치(신뢰/지역/예방/공동체)
  - 서비스 3종 카드 + “바로 신청” CTA
  - 조합원 혜택(참여/혜택/의사결정/프로그램 우선)
  - 실제 이용 흐름(3~5 step)
  - 후기/사례
  - FAQ
  - 문의 CTA + 연락 채널

- 조합 소개
  - 미션/비전/가치
  - 운영 원리(조합원 중심)
  - 투명성/신뢰 섹션(운영 원칙)

- 조합원 모집/가입 안내
  - 왜 조합원인가(혜택/가치)
  - 가입 절차(간단)
  - 가입 CTA(가입폼)
  - 로그인 시 가입 상태 확인

- 서비스 안내 허브(3 카테고리)
  - 재택의료 / 마을돌봄 / 건강공동체 프로그램
  - 각 상세 페이지: 대상/이용방법/비용(있으면)/준비물/FAQ/신청 CTA

- 커뮤니티(조합원 전용 글쓰기)
  - 목록/상세/글쓰기
  - 카테고리: 소식/건강정보/자유/후기
  - 신고 기능(옵션)

- 공지사항
  - 목록/상세
  - 공지 유형 배지(전체/조합원/긴급)
  - 첨부파일(옵션)

- 문의사항
  - 문의 작성(비회원도 가능: 답변 확인 방식은 이메일+비밀번호 또는 로그인 기반)
  - 로그인 사용자는 “내 문의” 목록/상세

- 로그인/회원가입
  - 로그인/비번 재설정
  - 조합원 가입(가입 신청과 계정 생성 관계를 UX에서 명확히)

3-2) 조합원 마이페이지
- 내 정보/조합원 상태
- 내 서비스 신청 내역(리스트) + 상세(진행 척도 타임라인)
- 내 커뮤니티 글/댓글
- 내 문의/답변

3-3) 관리자 운영 OS(Admin)
- 대시보드: 오늘/이번주 일정, 접수대기/상담중/배정필요/긴급 카운트,
  서비스별 신청 추이, 평균 처리시간/완료율
- 신청/접수 관리(핵심):
  - 탭: 캘린더 뷰 / 리스트 뷰
  - 캘린더: 일정 드래그 변경(가능하면), 이벤트 클릭 시 우측 상세 패널(신청정보/상태/담당/체크리스트)
  - 리스트: 필터(상태/서비스/지역/담당/날짜), 정렬(최신/긴급/방문일), 검색(이름/연락처/주소/키워드),
    일괄 처리(상태 변경/담당 배정)
- 서비스 카탈로그 관리: 생성/수정/노출/숨김, 서비스별 신청 폼 추가 질문 템플릿(가능하면)
- 공지 관리: 등록/수정/삭제, 유형(전체/조합원/긴급), 상단고정/예약게시(옵션)
- 문의 관리: 목록(상태/담당/우선순위), 답글 작성/수정/삭제, 템플릿 답변, 상태 변경(대기/처리중/완료)
- 커뮤니티 관리: 게시글/댓글 관리, 신고 처리, 숨김/삭제, 금칙어 필터(옵션)
- 계정/권한 관리: 관리자 계정/권한 부여, 조합원 상태(가입 승인/보류/탈퇴)

========================
4) UI/UX 원칙(의료·돌봄 톤)
========================
- 키워드: “따뜻함 + 신뢰 + 명확함”
- 과도한 테크/사이버 느낌 금지(강한 오렌지/레드 메인 금지)
- 고령층도 읽기 쉽게: 큰 폰트, 넉넉한 줄간격/여백, 버튼 크기 크게
- 상태 표시(접수/상담/배정/완료)를 “배지 + 타임라인/스텝퍼”로 시각화
- 모든 폼/테이블은 상태 UI 포함:
  로딩/성공/실패/권한없음/네트워크 오류/빈값(Empty)
- 제출 후 접수번호/내역 링크/알림을 제공

=========================================
5) 메인 히어로(WebGL Hero) 적용 + 색감/폰트 수정 요구
=========================================
중요: 메인 히어로는 “내가 제공한 React WebGL Hero 코드”를 개발 구현에 사용한다.
하지만 현재 코드의 오렌지/옐로/레드 색감과 폰트/조합은 의료사회적협동조합 톤과 맞지 않으니,
Figma 디자인과 개발 적용을 위해 아래와 같이 “따뜻한 돌봄/치유” 톤으로 재설계해라.

(1) 목표 팔레트(권장)
- Background: #071A2B (딥 네이비)
- Surface: #0B2A3A (딥 블루-그린)
- Warm highlight: #F6E7D8 (크림)
- Care mint: #BFE9E2 (민트)
- Sky: #CFE8FF (연하늘)
- Text: #F3F6FF (오프화이트)
- 경고/긴급: 레드 계열은 상태 배지에서만 제한적으로 사용

(2) 타이포그래피(한국어 최적)
- 기본 폰트: Pretendard (대체: Noto Sans KR / Apple SD Gothic Neo / system-ui)
- 제목: 700~800, 자간 -1% 정도(또렷하고 따뜻)
- 본문: 400~500, 줄간격 넉넉(1.6~1.8)
- 버튼/배지: 600

(3) 히어로 레이아웃/스타일 지시
- 배경은 WebGL 추상 “구름/안개” 느낌의 움직이는 배경이되,
  캔버스 위에 “따뜻한 안개 오버레이(그라데이션+블러)”를 반드시 적용해 가독성과 돌봄 무드를 강화한다.
- 헤드라인 그라데이션: 오렌지/레드 금지 → 크림/민트/스카이 조합으로 변경
- TrustBadge: 오렌지 계열 금지 → 화이트/민트/스카이 톤의 글래스/소프트 보더
- CTA 버튼:
  - Primary: 민트→스카이 그라데이션, 텍스트는 네이비 계열(신뢰)
  - Secondary: 투명/글래스 + 화이트 보더(차분)
- 히어로 문구/CTA 예시(디자인에 반영):
  - TrustBadge: “지역과 함께하는 재택의료 · 돌봄 · 건강공동체”
  - Headline line1: “당신의 일상에”
  - Headline line2: “따뜻한 돌봄을”
  - Subtitle: “횡성의료사회적협동조합은 방문진료·재택의료·마을 돌봄으로 지역의 건강한 삶을 함께 만들어갑니다.”
  - CTA Primary: “서비스 신청하기”
  - CTA Secondary: “조합원 가입하기”

(4) 구현 관점 요청(개발 힌트)
- WebGL shader 색을 직접 바꾸기 어렵다면, “캔버스 오버레이 레이어 + 텍스트/버튼 팔레트”로 톤을 맞추는 방식으로 설계해라.
- Tailwind 기반이라면 동적 클래스(text-${...})는 빌드에서 누락될 수 있으므로 고정 클래스로 설계해라.

================================
6) 상태/알림/자동화 요구
================================
- 서비스 신청 시:
  - 접수 완료 화면(접수번호/상태/예상 연락 시간)
  - 마이페이지에 자동 기록
- 관리자 OS:
  - 신청이 들어오면 “접수 대기”로 자동 분류
  - 일정 확정 시 캘린더 이벤트 자동 생성
  - 상태 변경/담당 배정/메모가 타임라인(로그)으로 누적
- 공지/문의/커뮤니티:
  - 등록/수정/삭제 권한 명확히
  - 관리자는 모든 콘텐츠 편집 가능
  - 문의 답변 시 사용자 알림 UX 포함(사이트 알림 또는 이메일)

================================
7) Figma Make 산출물 요구(구체)
================================
- 와이어프레임이 아니라, 실제 서비스 수준의 UI
- 데스크톱 우선 + 모바일 반응형(주요 화면)
- 레이아웃 템플릿 통일: Public(헤더/푸터), Member(마이페이지 레이아웃), Admin(사이드바+상단바)
- 컴포넌트화: 카드/배지/타임라인/스텝퍼/캘린더/테이블/필터바/폼/모달/슬라이드오버 상세패널
- Admin OS는 “매일 쓰는 도구”처럼 빠르고 즉시 처리 가능:
