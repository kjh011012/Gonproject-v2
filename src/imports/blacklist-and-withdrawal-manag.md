너는 시니어 제품 디자이너(UX/UI) + 운영OS 설계자 + 테이블 UX 전문가다.
현재 파일에는 “관리자(Admin) 웹앱”의 메뉴(사이드바)와 페이지 프레임(빈 화면)이 이미 생성되어 있다.
이번 작업의 목표는 “빈 페이지(본문 미작성)”로 남아 있는 메뉴의 페이지를 ‘운영 가능한 화면’으로 완성하는 것이다.

⚠️ 중요:
- 기존 Admin App Shell(좌측 사이드바/상단 탑바/브랜드 스타일/기본 프레임)은 유지한다. (재생성/재배치하지 말고, 빈 본문 영역만 채워라)
- 고객용 웹앱에서 사용한 디자인 시스템을 그대로 가져온다(브랜드 3색 + 중립색).
- 색상은 브랜드 3색만 사용하고, 상태/경고/위험을 표현하기 위해 새로운 빨강/주황 계열을 추가하지 않는다.
  → 경고/주의는 “아이콘 + 문구 + Accent(#F2EBDD) 배경 + 중립 텍스트/테두리”로 표현한다.
- 관리자 화면은 Desktop 중심(1440)이며, Mobile(390)은 “긴급 운영(검색/오늘 일정/문의 답변/조합원 검색)” 위주 축약 UI를 제공한다.
- 모든 데이터 화면(테이블)은 500건/일 규모에서도 운영 가능한 구조(검색/필터/저장뷰/벌크/우측 디테일 패널/스켈레톤/빈상태)를 기본 제공한다.
