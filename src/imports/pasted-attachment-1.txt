너는 제품디자이너 + UX라이터 + IA 설계자 + 접근성 전문가다.
목표는 “횡성의료사회적협동조합” 웹앱 홈페이지를 ‘이미지 우선(Image-first)’으로 재구성하여,
어르신(60대+)이 글을 거의 읽지 않아도 사진/구성만 보고 즉시 “이 조합은 이런 곳이구나 / 이런 서비스를 하는구나”를 이해하고,
바로 ‘조합원 가입(원큐)’ 또는 ‘전화 상담’으로 전환되게 만드는 것이다.

────────────────────────────────────────
0) 프로젝트 전제 / 유지 조건
────────────────────────────────────────
- 메인 히어로(홈 첫 화면)는 사용자가 이미 만들어 둔 것을 ‘그대로 유지’한다.
  - 히어로는 컴포넌트로 승격하고 “Lock(고정)” 처리한다.
  - 다만 모드(일반/어르신)에 따라 히어로 내부 텍스트 크기만 자동 적용되게 Variant로 만들 수 있다.
- 데스크톱(Web 브라우저)에서는 전체 레이아웃이 “현재 상태처럼 보이도록” 기본 구조/톤을 유지한다.
  - 즉, Desktop에서는 과한 재배치 대신 “이미지 우선 섹션 추가, 카드 디자인 교체, 콘텐츠 강화” 중심으로 업그레이드한다.
- 모바일에서는 완전히 모바일 친화적으로 재구성한다.
  - 한 화면에 정보 밀집 금지
  - 이미지 + 큰 제목 + 1줄 설명 + 큰 버튼(전화/가입) 중심으로 다시 배치한다.

1) 웹앱(App) 구조 / 정보구조(IA)
────────────────────────────────────────
- 웹앱 형태로 구성한다(SPA처럼 느껴지는 App Shell).
- 공통 App Shell
  - Desktop/Tablet: 상단 고정 Header + 좌측 로고 + 메뉴 + 우측 CTA + 모드 토글
  - Mobile: 상단 얇은 Header + 하단 고정 Bottom Navigation(어르신모드에서는 항상 표시)
- 페이지 라우팅(프레임/스크린 생성)
  1) Home (/)              - Normal / Senior + Desktop/Mobile
  2) About (/about)        - Normal / Senior + Desktop/Mobile
  3) Services (/services)  - Normal / Senior + Desktop/Mobile
     - 서비스 상세는 페이지 이동 대신 “아코디언 확장” 또는 “Bottom Sheet(모바일)”로 열린다.
  4) Join (/join)          - Normal / Senior + Desktop/Mobile (원큐 가입 3-Step)
  5) Community (/community)- Normal / Senior + Desktop/Mobile (공지/모임/자료)
  6) Contact (/contact)    - Normal / Senior + Desktop/Mobile (전화/카톡/문의/지도)
- 어디서든 전환(CTA)이 끊기지 않게:
  - 상단(Desktop) ‘조합원 가입하기’ Primary CTA 고정
  - 모바일(특히 어르신모드) 하단 고정 CTA: [서비스][가입][공지][전화]

────────────────────────────────────────
2) 디자인 원칙(가장 중요): 이미지 우선(Image-first)
────────────────────────────────────────
[핵심 규칙]
- 전체 사이트를 “이미지 우선(Image-first)”으로 재구성한다.
- 모든 핵심 섹션(조합 소개/서비스/가입)은 텍스트보다 이미지가 먼저 보이게 설계한다.
- 각 서비스는 ‘서비스 포스터 카드’로 표현한다:
  - 카드 상단 60~70%는 이미지(사진 placeholder)
  - 카드 하단에 큰 제목(짧게) + 1줄 설명 + CTA 1개
- 어르신모드에서 서비스/상황 선택은 “이미지 타일(버튼처럼 클릭)”로 구성한다.

[About 페이지 구성 필수 포함]
1) 단체/현장 느낌 대표 이미지 1장
2) 상황 타일 6개(걷기힘듦/약관리/혼자불안/퇴원후회복/돌봄연결/예방)
3) 서비스 포스터 5~6개
4) 3컷 스토리보드(전화→상담→연결/방문)
5) 신뢰 섹션(사람 사진 + 배지 3개)
6) 출자금(씨앗돈) 비유 카드(일러스트 1개)

[Services 페이지 구성 필수 포함]
- 상단에 “내 상황 고르기” 이미지 타일 6개
- 아래에 서비스 포스터 6개 그리드
- 각 포스터 클릭하면 상세(이런 분께/무엇을/이용방법/준비물/문의)가 아코디언으로 펼쳐지게 한다.
  - Mobile은 아코디언 대신 Bottom Sheet로도 가능(더 읽기 쉬움)

[Join 페이지 구성 필수 포함]
- 상단에 ‘전화 도움’ 이미지를 넣고 큰 버튼(전화로 가입 도와주세요)을 고정
- 가입 1-2-3 단계는 큰 숫자 + 아이콘/일러스트로 설명한다
- 출자금 설명은 “씨앗돈” 비유 일러스트 카드로 시각화한다

[이미지 위 텍스트 가독성 규칙]
- 브랜드 컬러 3개(Primary #1F6B78, Secondary #67B89A, Accent #F2EBDD) + 중립색만 사용
- 사진 위에는 검정(#111827) 투명 그라데이션 오버레이만 허용(새 컬러 추가 금지)
- 어르신모드에서 제목은 이미지 위에 크게, 설명은 이미지 아래 1줄만

[이미지 Placeholder 라벨 규칙]
- 모든 이미지 placeholder에는 라벨을 넣는다(텍스트로 표시).

8) 모바일 전용 재구성 규칙(웹앱 모바일 최적화)
────────────────────────────────────────
- 모바일에서는 Desktop 섹션을 그대로 축소하지 말고 “모바일 전용 섹션 순서”로 재배치한다.
- Mobile Home 권장 순서:
  1) (기존 히어로 유지) + 하단 CTA(전화/가입) 2버튼
  2) 상황 타일 6개(내가 겪는 문제부터 시작)
  3) 서비스 포스터 4개(핵심만) + 더보기(전체 서비스)
  4) 3컷 스토리보드(전화→상담→연결)
  5) 공지/모임 미리보기 3개
  6) 문의(전화/카톡/문의)
- Mobile 어르신모드에서는 하단 BottomNav 고정:
  - [서비스] [가입] [공지] [전화]
- 모바일 카드 규칙:
  - 이미지가 위, 텍스트는 아래(1줄 설명까지만)
  - 상세는 아코디언/Bottom Sheet로 열기

9) 프로토타입 인터랙션(클릭 흐름)

- 상황 타일 클릭:
  - Services 페이지로 이동하면서 해당 상황이 선택된 상태로 스크롤/필터 적용
- 서비스 포스터 클릭:

- 가입 CTA 클릭:
  - Join 페이지로 이동 + Step 1으로 포커스
- 전화 버튼:
